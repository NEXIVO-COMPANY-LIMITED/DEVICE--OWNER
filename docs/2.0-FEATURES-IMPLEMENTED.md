- ‚úÖ Foreground service for reliability

### Heartbeat Payload
```json
{
  "device_id": "[DEVICE_ID]",
  "timestamp": "[TIMESTAMP]",
  "device_info": {
    "imei": "[DEVICE_IMEI]",
    "model": "[DEVICE_MODEL]",

    "battery_level": [BATTERY_LEVEL],
    "storage_free": [STORAGE_FREE]
  },
  "security_status": {
    "developer_options": false,
    "usb_debugging": false,
    "device_owner_active": true,
    "tamper_detected": false
  },
  "location": {
    "latitude": [LATITUDE],
    "longitude": [LONGITUDE],
    "accuracy": [ACCURACY]
  }
}
```

### Server Commands
| Command | Action | Implementation |
|---------|--------|----------------|
| `lock` | Apply hard lock | `RemoteDeviceControlManager` |
| `unlock` | Remove lock | `RemoteDeviceControlManager` |

| `update_config` | Update settings | Configuration manager |

### Key Components

## 4. Lock Mechanisms

### Hard Lock (Kiosk Mode)
- ‚úÖ Full-screen kiosk lock
- ‚úÖ Cannot be dismissed by user
- ‚úÖ Persists across reboots
- ‚úÖ Blocks all device functions
- ‚úÖ Used for overdue payments and tamper

### Soft Lock (Overlay)
- ‚úÖ Dismissible overlay
- ‚úÖ Shows payment information
- ‚úÖ User can still use device
- ‚úÖ Used for payment reminders
- ‚úÖ SIM change warnings

### Lock Triggers
| Trigger | Type | Reason |
|---------|------|--------|
| Payment Overdue | Hard | Payment not received |
| Payment Reminder | Soft | Payment due soon |
| Tamper Detected | Hard | Security violation |
| SIM Change | Soft | SIM card changed |
| Server Command | Hard/Soft | Remote management |

### Key Components
- `HardLockGenericActivity.kt` - Hard lock UI
- `SoftLockOverlayService.kt` - Soft lock overlay
- `RemoteDeviceControlManager.kt` - Lock management
- `HardLockManager.kt` - Hard lock logic

### Lock Persistence
- Stored in SharedPreferences
- Device-protected storage
- Survives app restart
- Survives device reboot

## 5. Tamper Detection

### Detection Methods

#### Boot-Time Checks
- ‚úÖ Developer options enabled
- ‚úÖ USB debugging enabled
- ‚úÖ Bootloader unlock status
- ‚úÖ Root detection

#### Runtime Monitoring
- ‚úÖ Developer options (every 2s)
- ‚úÖ USB debugging (every 2s)
- ‚úÖ Settings app foreground (every 500ms)
- ‚úÖ Device Owner status (every 5s)

#### Event-Based Detection
- ‚úÖ SIM card changes
- ‚úÖ App removal/replacement
- ‚úÖ Unauthorized accessibility services
- ‚úÖ Package installation

### Tamper Types
| Type | Detection | Response |
|------|-----------|----------|
| Developer Options | Runtime check | Hard lock + report |
| USB Debugging | Runtime check | Hard lock + report |
| Bootloader Unlock | Boot check | Hard lock + report |
| Root Detection | File system | Hard lock + report |
| SIM Change | Telephony | Soft lock + report |
| App Removal | Package monitor | Hard lock + report |
| Device Owner Removal | Periodic check | Hard lock + wipe |

### Response Flow
```
Tamper Detected
    ‚Üì
Apply Hard Lock
    ‚Üì
Log Event Locally
    ‚Üì
POST /api/device/tamper
    ‚Üì
If Offline: Queue for Sync
```

### Key Components
- `EnhancedTamperDetector.kt` - Main detector
- `TamperBootChecker.kt` - Boot-time checks
- `EnhancedAntiTamperResponse.kt` - Response handler
- `SecurityMonitorService.kt` - Runtime monitoring
- `SIMChangeDetector.kt` - SIM monitoring
- `DeviceOwnerRemovalDetector.kt` - Device Owner monitoring

### API Endpoint
```
POST /api/device/tamper
```
