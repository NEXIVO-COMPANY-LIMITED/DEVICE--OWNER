# 12.0 AGENT UPDATE (AUTO-UPDATE)

This document describes **automatic app updates** from GitHub Releases: how the app checks for new versions, downloads the APK, and installs silently using Device Owner privileges.

---

## Overview

The Device Owner app can **update itself** without user interaction by:

1. Fetching the latest release from a **GitHub** repository.  
2. Downloading the release APK.  
3. Installing it silently via **PackageInstaller** (Device Owner / device admin can install packages without user approval).  
4. Optionally notifying the user (e.g. "Update available", "Downloading", "App up to date").

## Components

### UpdateConfig

**Class**: `com.microspace.payo.update.config.UpdateConfig`

**Purpose**: Configuration constants for the update system.

| Constant | Purpose |
|----------|---------|
| GITHUB_OWNER | GitHub org/user (e.g. NEXIVO-COMPANY-LIMITED) |
| GITHUB_REPO | Repository name (e.g. DEVICE--OWNER) |
| GITHUB_API_URL | `https://api.github.com/repos/{owner}/{repo}/releases/latest` |
| APK_FILE_NAME | Asset name in release (e.g. app-release.apk) |
| UPDATE_CHECK_INTERVAL_HOURS | Interval for periodic check (e.g. 6 hours) |
| INITIAL_DELAY_MINUTES | Delay before first check (e.g. 30 minutes) |
| WORK_NAME | WorkManager unique work name (e.g. github_update_check) |
| NOTIFICATION_CHANNEL_ID | Update notification channel |
| NOTIFICATION_ID | Update notification ID |
| AUTO_INSTALL_UPDATES | Whether to install automatically when update found |
| REQUIRE_WIFI | Whether to require WiFi for download |
| REQUIRE_CHARGING | Whether to require charging |
| ALLOW_DOWNGRADE | Whether to allow installing older version |
| ACTION_INSTALL_COMPLETE | Broadcast action when install finishes |

### GitHubUpdateManager

**Class**: `com.microspace.payo.update.github.GitHubUpdateManager`

**Purpose**: Manages the update check and installation process.

#### Key Methods:

**checkAndUpdate()**
- Gets current app version
- Fetches latest release from GitHub API
- Compares versions; if update available → **downloadAndInstallUpdate(latestRelease)**
- Shows notifications:
  - "Update available"
  - "Downloading..."
  - "App up to date"
  - "Update error"

**fetchLatestRelease()**
- HTTP GET to `UpdateConfig.GITHUB_API_URL`
- Parses JSON response
- Returns release information

**parseGitHubRelease(json)**
- Extracts version tag from release
- Finds APK download URL from release assets
- Returns structured release data

**downloadAndInstallUpdate(release)**
- Downloads APK to internal storage
- Uses **PackageInstaller** to install silently (Device Owner privilege)
- Broadcasts **ACTION_INSTALL_COMPLETE** when done

**Version Comparison:**
```kotlin
// Example version comparison
currentVersion = "1.0.5"
latestVersion = "1.1.0"
// Result: Update available
```

---

### UpdateScheduler

**Class**: `com.microspace.payo.update.scheduler.UpdateScheduler`

**Purpose**: Schedules periodic update checks using WorkManager.

#### Key Methods:

**schedulePeriodicChecks(context)**
- Creates periodic WorkRequest for **UpdateCheckWorker**
- Sets interval from **UpdateConfig.UPDATE_CHECK_INTERVAL_HOURS**
- Sets constraints (WiFi, charging if configured)
- Enqueues unique work with **KEEP** policy

**Example:**
```kotlin
// Called from DeviceOwnerApplication
if (DeviceOwnerManager(this).isDeviceOwner()) {
    UpdateScheduler.schedulePeriodicChecks(this)
}
```

---

### UpdateCheckWorker

**Class**: `com.microspace.payo.update.worker.UpdateCheckWorker`

**Purpose**: WorkManager worker that performs periodic update checks.

**Execution:**
1. Worker runs at scheduled interval
2. Calls **GitHubUpdateManager.checkAndUpdate()**
3. Returns success/failure result
4. WorkManager reschedules automatically

**Constraints:**
- Network required (WiFi if configured)
- Battery not low
- Device idle (optional)

---

### UpdateReceiver

**Class**: `com.microspace.payo.update.receiver.UpdateReceiver`

**Purpose**: Broadcast receiver for update completion events.

**Actions Handled:**
- **ACTION_INSTALL_COMPLETE**: Install finished successfully
- **ACTION_ROLLBACK_COMPLETE**: Rollback finished (if implemented)

**Post-Install Logic:**
- Cancel update notification
- Log success to backend
- Clear update cache
- Restart services if needed

## Update Flow

### 1. Startup Check
```
DeviceOwnerApplication.onCreate()
    ↓
Check if Device Owner
    ↓
UpdateScheduler.schedulePeriodicChecks(context)
    ↓
Delay 30 minutes
    ↓
GitHubUpdateManager.checkAndUpdate() (first check)
```

### 2. Periodic Check
```
UpdateCheckWorker (every 6 hours)
    ↓
GitHubUpdateManager.checkAndUpdate()
    ↓
Fetch latest release from GitHub
    ↓
Compare versions
    ↓
If newer → Download and install
```

### 3. Download and Install
```
downloadAndInstallUpdate()
    ↓
Show "Downloading..." notification
    ↓
Download APK to internal storage
    ↓
Verify APK signature
    ↓
PackageInstaller.install() (silent)
    ↓
Broadcast ACTION_INSTALL_COMPLETE
    ↓
UpdateReceiver handles completion
```

## GitHub Release Format

### Required Release Structure:

```json
{
  "tag_name": "v1.1.0",
  "name": "Version 1.1.0",
  "body": "Release notes...",
  "assets": [
    {
      "name": "app-release.apk",
      "browser_download_url": "https://github.com/.../app-release.apk",
      "size": 15728640
    }
  ]
}
```

### Version Tag Format:
- **Recommended**: `v1.0.0`, `v1.1.0`, `v2.0.0`
- **Also supported**: `1.0.0`, `1.1.0`, `2.0.0`
- **Pattern**: `v?MAJOR.MINOR.PATCH`

## Permissions Required

### Manifest Permissions:

```xml
<!-- Auto-update permissions -->
<uses-permission android:name="android.permission.REQUEST_INSTALL_PACKAGES" />
<uses-permission android:name="android.permission.INSTALL_PACKAGES" 
    tools:ignore="ProtectedPermissions" />
<uses-permission android:name="android.permission.DELETE_PACKAGES" 
    tools:ignore="ProtectedPermissions" />

<!-- Network for GitHub API -->
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />

<!-- Notifications (Android 13+) -->
<uses-permission android:name="android.permission.POST_NOTIFICATIONS" />
```

### Runtime Permissions:
- **POST_NOTIFICATIONS**: Requested at runtime on Android 13+
- **INSTALL_PACKAGES**: Granted automatically when Device Owner

## Notification System

### Update Notification States:

| State | Title | Message | Action |
|-------|-------|---------|--------|
| **Checking** | Checking for updates | Please wait... | None |
| **Available** | Update available | Version X.X.X is ready | Install |
| **Downloading** | Downloading update | X% complete | Cancel |
| **Installing** | Installing update | Please wait... | None |
| **Complete** | Update installed | App updated to X.X.X | OK |
| **Error** | Update failed | Error message | Retry |
| **Up to date** | No updates | You have the latest version | OK |

### Notification Channel:
- **ID**: `update_channel`
- **Name**: "App Updates"
- **Importance**: HIGH (shows heads-up)
- **Sound**: Default notification sound

## Error Handling

### Common Errors:

| Error | Cause | Solution |
|-------|-------|----------|
| **Network error** | No internet connection | Retry when online |
| **404 Not Found** | No releases in repository | Create a release on GitHub |
| **Invalid APK** | Corrupted download | Re-download |
| **Signature mismatch** | APK signed with different key | Use same signing key |
| **Install failed** | Not Device Owner | Ensure Device Owner status |
| **Parse error** | Invalid JSON response | Check GitHub API format |

### Error Logging:
- All errors logged to **ServerBugAndLogReporter**
- Includes error message, stack trace, and context
- Sent to backend for monitoring

## Testing

### Manual Update Test:

1. **Create GitHub Release:**

## Security Considerations

### APK Verification:
1. **Signature Check**: Verify APK signed with same certificate
2. **Checksum Validation**: Compare SHA-256 hash (if provided in release)
3. **HTTPS Only**: All downloads over HTTPS
4. **Source Validation**: Only from configured GitHub repository

### Update Safety:
- **Rollback Support**: Keep previous APK for rollback
- **Backup Data**: Backup critical data before update
- **Staged Rollout**: Test on subset of devices first
- **Kill Switch**: Backend can disable auto-update
